Proyectos tecnológicos

MUSK-XGBoost: Clasificador de Cáncer de Alto Rendimiento basado en Transfer Learning Multimodal

Este proyecto implementa una arquitectura de Machine Learning de dos fases (Transfer Learning) para la clasificación de imágenes de patología (WSI), diferenciando entre tejido Normal y Tumor. Utiliza el modelo fundacional MUSK (Multimodal transformer with Unified maSKed modeling), preentrenado en millones de datos de patología, como extractor de features de alta calidad, y el algoritmo XGBoost como clasificador final de alto rendimiento. El enfoque no solo prioriza el rendimiento de Estado del Arte (SOTA), sino también la Explicabilidad (XAI) de las predicciones, utilizando herramientas como LIME para aumentar la confianza clínica.

El desarrollo de este clasificador de oncología de precisión consolidó mi experiencia en el uso de Modelos Fundacionales y Transfer Learning en Medicina, aplicando la arquitectura de dos fases (MUSK como extractor de features de alta calidad y XGBoost como clasificador) esencial para el manejo de datos médicos escasos. Además, demostré Validación Rigurosa y MLOps Básico al implementar la Validación Cruzada Estratificada y utilizar métricas avanzadas (AUC, F1-Score), estableciendo líneas base (Regresión Logística) y comparadores SOTA (LightGBM) para justificar el rendimiento del modelo propuesto. Crucialmente, mi enfoque priorizó la Explicabilidad (XAI) Orientada al Cliente Clínico, entendiendo que la Matriz de Confusión debe traducir el rendimiento técnico (ej., la reducción de 117 a 18 Falsos Negativos) en valor clínico (minimización del riesgo de diagnóstico tardío). Finalmente, la Optimización de Recursos y Rendimiento se abordó mediante un análisis comparativo de la eficiencia computacional entre XGBoost y LightGBM, un factor clave para la selección del modelo más adecuado para su implementación en entornos de producción con limitaciones de latencia.

TumorView: Visor y Evaluador de Imágenes Patológicas para Deep Learning

La aplicación TumorView es un sistema web desarrollado en Flask (Python) diseñado para la curación de imágenes médicas (catalogación, anotación y seguimiento), facilitando el trabajo de múltiples usuarios o evaluadores en el área de Patología Digital. Su propósito principal es proporcionar una interfaz eficiente para patólogos y científicos, permitiéndoles la revisión visual de imágenes de diferentes pacientes y su clasificación en categorías (por ejemplo, tumor, normal o indeterminado) para el desarrollo posterior de modelos de clasificación basados en Deep Learning.

El principal desafío técnico en este proyecto fue implementar un sistema de anotación y seguimiento de progreso que fuera ligero, multiusuario y persistente, sin depender de una base de datos compleja. La solución se centró en la creación de una arquitectura de backend con Python/Flask que gestiona los datos de curación utilizando JSON para el almacenamiento por usuario, lo que demostró mi capacidad para diseñar soluciones escalables y funcionales en entornos con restricciones de infraestructura. Este trabajo consolidó mis habilidades en el desarrollo full-stack enfocado en la usabilidad de herramientas científicas, mejorando significativamente la eficiencia en la etapa crítica de curación de datos para Deep Learning en patología digital.

Pipeline Automatizado de Producción: Modelo MLOps aplicado en cáncer de mama

MLOps: Pipeline Automatizado de Producción. Este proyecto es una demostración integral y práctica de un pipeline completo de MLOps (Machine Learning Operations), diseñado para llevar modelos de Machine Learning desde la fase experimental hasta la producción de manera automatizada, reproducible y escalable. Se utilizó una arquitectura modular y herramientas líderes en la industria para establecer un flujo de Integración Continua / Despliegue Continuo (CI/CD) que minimiza la fricción entre la ciencia de datos y las operaciones de ingeniería.

Este proyecto fue crucial para transicionar de la experimentación a la ingeniería de producción. El aprendizaje fundamental fue la orquestación efectiva de un flujo de Integración Continua / Despliegue Continuo (CI/CD). La implementación de DVC garantizó la trazabilidad de los datasets y modelos, mientras que MLflow permitió un seguimiento transparente de las métricas de rendimiento. El uso de Docker y GitHub Actions me permitió automatizar el deployment del modelo como un servicio de API, validando mi conocimiento en DevOps aplicado a ML. Este pipeline asegura que el modelo pueda ser re-entrenado y re-desplegado de manera confiable, un pilar esencial para llevar la investigación traslacional a la práctica clínica.
